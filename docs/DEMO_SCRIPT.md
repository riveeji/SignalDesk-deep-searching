# Demo Script

## 2 分钟版本

### 1. 开场

“这个项目叫 `SignalDesk / 研策台`。  
它不是聊天壳子，而是一个面向 AI / Agent 技术选型的 `Deep Research Agent`。”

### 2. 首页

打开首页，直接输入问题：

`如果团队是 Python 技术栈，想做可观测、可接管、适合生产环境的 Agent 系统，LangGraph、PydanticAI 和 Mastra 哪个更适合作为底座？`

强调：

- 首页只有一个搜索入口
- 用户不需要先手动配置流程
- 系统会自己澄清、规划、搜证、综合

### 3. 任务详情

打开完成的 run，讲三块：

- `已锁定范围`：候选项和比较维度
- `证据覆盖矩阵`：每个候选项有没有官方、仓库、外部/学术来源
- `来源池 + 步骤轨迹`：可排除来源、可回放步骤、可阶段级重跑

### 4. 报告页

重点强调：

- 不是摘要，而是结构化推荐报告
- 有 `verdict / confidence / missing_evidence`
- 每个核心判断都有 citation
- 可以导出 Markdown 和 PDF

### 5. 收尾

“这个项目最重要的不是会写报告，而是会在证据不足时主动降级，不强行给答案。”

## 5 分钟版本

### 1. 业务背景

“很多 AI 项目只会聊天，但真实技术选型需要先收敛问题、再系统搜证、最后才能比较和决策。我想做的是一个研究产品，而不是一个问答页面。”

### 2. 核心流程

依次讲：

1. `Clarifier`
   把模糊问题改造成结构化研究范围
2. `Planner`
   生成研究计划树
3. `Retriever`
   聚合 GitHub、官方文档、网页和学术元数据
4. `Synthesizer`
   输出报告草稿
5. `Answer Validator`
   检查是否对题、是否覆盖完整、能不能给强结论

### 3. 工程亮点

- PostgreSQL 持久化 `Run / Plan / Source / Step / Citation / Report`
- 可插拔 `SearchProvider`
- 手动导入来源
- 来源排除后自动重新综合
- PDF 导出带图表

### 4. 关键演示点

优先演示两个场景：

- 正常选型题：`research_22f8fcc3c7`
- 证据不足降级：`research_0a1068cdbb`

第二个场景要明确说明：

- 人工排除 Mastra 关键来源后
- 系统自动把结论降成 `insufficient_evidence`
- 同时写清楚缺了哪些证据

### 5. 结束

“这个项目让我能同时讲产品价值、Agent 流程、后端编排、持久化设计和模型输出约束，而不只是讲一个前端页面。”

## 建议演示顺序

1. 首页输入问题
2. 打开一个已完成 run
3. 讲覆盖矩阵和来源池
4. 打开报告页
5. 演示证据不足降级

## 不建议在 demo 里讲太久的内容

- 不要一上来讲数据库表结构
- 不要先讲技术栈，再讲产品
- 不要把它讲成“我做了一个聊天机器人”
- 不要只展示漂亮页面，不解释 `verdict / confidence`
