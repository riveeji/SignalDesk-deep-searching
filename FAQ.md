# 常见问题

## 1. 这个项目和普通 RAG 有什么区别？

普通 RAG 通常是“检索后直接回答”。  
`SignalDesk` 会先做问题澄清，再生成研究计划，再多源搜证，最后综合并校验结果质量。

## 2. 为什么要做问题澄清？

因为很多技术选型问题天然是模糊的。  
如果不先收敛范围，后面的检索和结论很容易答非所问。

## 3. 为什么要支持人工介入？

真实研究流程里，范围澄清和来源取舍本来就应该允许人参与。  
这也是减少错误结论的重要方式。

## 4. 为什么不默认自动抓取 Google Scholar 或知网？

主要是稳定性和合规性问题。  
当前策略是保留手动导入入口，但不默认自动抓取。

## 5. 这个项目更偏 AI 还是更偏工程？

两者都有，但更偏 `AI system / agent engineering`。  
重点不是训练模型，而是让模型参与一个可解释、可约束、可追溯的研究流程。

## 6. 它适合用来做什么？

- AI / Agent 框架选型
- 推理后端评估
- 开源项目对比研究
- 团队内部技术调研

## 7. 当前最值得继续做的方向是什么？

- 提升检索覆盖稳定性
- 改善结果质量约束
- 增强报告可视化
- 完善部署与演示体验
